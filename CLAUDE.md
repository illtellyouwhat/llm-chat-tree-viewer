## What This Is

A Chrome sidebar extension (MV3) that renders LLM conversations as navigable node trees.
Each conversation turn is a node. Clicking a node navigates to that turn. Users can annotate
turns (star, color, notes), mark turns as unimportant, and export the annotated tree.

**The sidebar is the surface. This is not a canvas application.**

---

## Tech Stack

- Chrome Extension Manifest V3
- Vanilla JS — no build step, no bundler, no framework
- No server-side infrastructure — Chrome extension only
- API keys stored in `chrome.storage.local`
- Platforms: ChatGPT (live), Claude.ai (next), Gemini (planned)

## Architecture: The `initPanel` Adapter Pattern

**This pattern is canonical. It must be preserved in all future work.**

| File | Role |
|------|------|
| `panel-core.js` | Shared sidebar logic: tree parsing, layout, rendering, annotations, prefs, settings |
| `content.js` | ChatGPT adapter: DOM scraping, sidebar injection, SPA navigation |
| `bridge.js` | Runs in MAIN world. Fires React click events for branch navigation |
| Future adapters | Follow the same pattern — `initPanel` call, adapter-specific DOM logic only |

**Platform differences live in adapter files only. Never move platform logic into `panel-core.js`.**

## Locked Architecture Decisions

These are decided. Do not relitigate without explicit user instruction.

| Decision | Constraint |
|----------|-----------|
| Export targets | Markdown and Obsidian Canvas only. No Miro, no Excalidraw, no other canvas tool. |
| LLM summaries | Via OpenRouter — stubbed. Not in scope until settings UI is designed. |
| Text drag-to-note | Cross-frame. Content script `dragstart` → message → sidebar. Not a one-liner. |
| Platform adapters | Independent. DOM analysis session required before each. Do not assume ChatGPT maps to either. |
| No server | Chrome extension only. No backend. API keys in `chrome.storage.local`. |
| No Supabase sync | Out of scope. Future consideration only. |

## Critical Rules

1. **Fix and extend, not rebuild.** Working code is the foundation. New capabilities layer on top.
   Do not rewrite working code to clean it up.

2. **U1 before N1/N2/N3.** Node text expansion (U1) replaces the tooltip system. Features that
   depend on node hover behavior must be built after U1, not before.

3. **DOM analysis before adapter code.** Do not write a Claude.ai or Gemini adapter without a
   real DOM inspection session. Do not assume the ChatGPT adapter maps to either.

4. **Platform-agnostic features verified on ChatGPT first** before touching adapters.

5. **Advisor writes specs; executor implements them.** Coordination through `current-phase/` only.
   Advisor and executor do not cross-talk.

6. **No cross-contamination from other projects.** The Supabase/archive project is a separate
   system. Do not import decisions or assumptions from it.

7. **Executor never enters `advisor/`.** That boundary is structural.
